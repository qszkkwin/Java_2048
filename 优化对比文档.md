# 2048 小游戏 — 重构优化对比文档

```
test_2048/
├── src/com/qsz/
│   ├── demo1/                          # 原始版本（未改动）
│   │   ├── MainFrame.java              # 538 行，上帝类
│   │   └── testMainFrame.java          # 入口
│   └── demo3/                          # 优化版本（新增）
│       ├── GameConstants.java          # [新增] 常量配置
│       ├── GameLogic.java              # [新增] 游戏逻辑层
│       ├── GameBoardPanel.java         # [新增] 自定义绘制面板
```

## 改动一览

### 1. 分离游戏逻辑 — 新增 GameLogic.java

| 原 MainFrame (demo1) | 优化后 (demo3) |
|---|---|
| `data[][]`、`score`、`loseFlag` 声明在 MainFrame 中 | 封装在 `GameLogic` 内部，对外只暴露 getter |
| `moveToleft(int flag)` 用 `flag=1/2` 区分是否计分 | `moveLeft()` 返回 `boolean`，得分自动累计 |
| `checkLeft/Up/Right/Down` 各拷贝一次 4×4 数组 → 移动 → 比较 → 恢复 | `checkGameOver()` 单次遍历，零拷贝 |
| `generatorNum()` 用固定数组存空位 | `generateTile()` 直接遍历，引入 10% 概率出 4 |
| `copyArray`、`reverseArray`、`reverse2DRowsInPlace` 等辅助方法 | 全部移除，不再需要 |
| 旋转用 4 步矩阵变换（transpose + swap + reverse + swap） | 坐标映射公式 `(i,j)→(j,3-i)`，2 行代码完成 |

**文件行数对比**：游戏逻辑部分从原 ~250 行精简为 ~130 行。

### 2. 自定义绘制 — 新增 GameBoardPanel.java

| 原 MainFrame.paintView() | 优化后 GameBoardPanel.paintComponent() |
|---|---|
| `getContentPane().removeAll()` | 直接 `repaint()`，无组件操作 |
| `new JLabel(new ImageIcon(...))` × 17+ | `g2d.drawImage()`，零对象创建 |
| 每次按键创建销毁 17+ Swing 组件，可能闪烁 | 图片一次缓存重复使用，双缓冲无闪烁 |
| 不支持无图片运行 | 纯色回退模式，无素材也能正常显示 |

### 3. 集中管理常量 — 新增 GameConstants.java

| 原 MainFrame | 优化后 |
|---|---|
| `setSize(514, 538)` — 直接写在代码中 | `GameConstants.WINDOW_WIDTH` |
| `setBounds(50 + 100 * i, ...)` — 魔法数字 | `GameConstants.GRID_X/GRID_Y/CELL_SIZE` |
| `keyCode == 37` — 含义不明 | 在 MainFrame 中用 `KeyEvent.VK_LEFT` 转换 |
| `"E:\\2048素材\\" + ... + ".png"` — 绝对路径 | `GameConstants.IMAGE_BASE` + 皮肤子目录 |

### 4. 优化 MainFrame.java

| 维度 | demo1/MainFrame.java | demo3/MainFrame.java |
|---|---|---|
| **行数** | 538 行 | ~120 行 |
| **职责** | 数据 + 逻辑 + 绘制 + 控制，全部揉在一起 | 只做窗口/菜单初始化和事件转发 |
| **数据** | `int[][] data` 直接暴露 | 委托 `GameLogic` 管理 |
| **绘制** | `paintView()` 自己画 | 委托 `GameBoardPanel` |
| **键盘码** | 硬编码 `37/38/39/40` | `KeyEvent.VK_LEFT/UP/RIGHT/DOWN` |
| **游戏结束判断** | 检查在 `move + generate` 之后 | ✅ 同上（修复了此前版本检查在 generate 之前的 bug） |
| **换肤** | 空方法，只打印日志 | 调用 `panel.switchSkin()` 实际切换 |

### 5. 图片路径优化

原版：
```java
new ImageIcon("E:\\2048素材\\" + data[j][i] + ".png");
```
问题：绝对路径，换电脑必须改代码。

优化版：
```java
// GameConstants.java
public static final String IMAGE_BASE = "assets/images/";

// GameBoardPanel.java — 根据当前皮肤拼接路径
String path = IMAGE_BASE + skinDir + "/" + imageName + ".png";
// 例如: assets/images/neon/background.png
```
优势：相对路径，支持多皮肤子目录，支持 classpath 加载。

### 6. 换肤功能

菜单项 → 调用 `GameBoardPanel.switchSkin("neon")` → 清空图片缓存 → 从新目录加载 → `repaint()`。

素材目录结构：
```
assets/images/
├── classic/    背景.png, 2.png, 4.png, ..., 2048.png, gameOver.png
├── neon/       同上（不同配色）
└── candy/      同上（不同配色）
```

## 操作步骤

1. **运行原版**：执行 `com.qsz.demo1.testMainFrame`
2. **运行优化版**：执行 `com.qsz.demo3.App`
3. **对比代码**：对比 `demo1/MainFrame.java` vs `demo3/MainFrame.java`
4. **换肤**：优化版菜单「换肤」→ 经典/霓虹/糖果

---

总体的优化方面：
1. **MVC 分离**：从单文件 538 行拆为 5 个文件，每层职责单一
2. **消除魔法数字**：所有常量归入 `GameConstants`，方向键用枚举/常量
3. **paintComponent**：替换 `removeAll+JLabel`，零对象创建，无闪烁
4. **算法优化**：游戏结束判断从 4 次数组拷贝降为 1 次 O(n²) 遍历
5. **策略可扩展**：图片路径从绝对路径改为相对路径 + 皮肤子目录，新增主题只需加目录
